# Contributing

This repository is maintained as a **public Railway-first template**.

## Contribution policy

By default, this repo accepts **dependency-update PRs only**.

If a code or documentation PR was explicitly requested by the maintainer, say that clearly in the PR description.

Unrequested feature/fix PRs may be declined even if technically correct. This is intentional: the template is maintained with a narrow product direction and low review overhead.

## Before opening a PR

1. Make sure the change fits the Railway-first direction of the repo.
2. Keep the diff focused.
3. Update docs if setup, deployment, or behavior changes.
4. Run relevant local checks.

## Development docs

- Local development guide: [docs/development.md](./docs/development.md)
- Railway quickstart: [docs/railway-quickstart.md](./docs/railway-quickstart.md)
- Full deployment guide: [docs/deployment.md](./docs/deployment.md)

## AI / automated contributions

Automation is welcome for maintainer workflows and dependency upkeep.

For external contributors, do not submit low-context AI-generated PRs that create more review work than implementation value. If the maintainer has to reconstruct the intent from scratch, the PR is not ready.

## Dependency PRs

Dependency-update PRs are the normal exception to the default PR policy. Those are reviewed for compatibility, deployment safety, and maintenance value.
